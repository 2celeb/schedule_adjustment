# プロジェクト解析レポート (doc/ana.md)

## 1. プロジェクト概要
本プロジェクトは、Discordコミュニティ（最大20名程度の小規模グループ）向けの**「ゆるい識別方式」を採用したスケジュール調整ツール**である。既存のツール（Doodle, 調整さん等）よりもDiscord連携とGoogleカレンダー同期に特化し、導入の心理的・技術的障壁を極力排除することを目標としている。

### 主要ターゲット
- Discordで活動する小規模なコミュニティ（ゲームギルド、勉強会、サークル等）。
- 定期的な活動日を調整・確定させる必要があるグループ。

---

## 2. コアコンセプト
1.  **ゆるい識別方式 (No Password)**
    - パスワード管理の煩わしさを排除。
    - 共通URLにアクセスし、名前を選択するだけで入力開始。
    - localStorageによるデバイス記憶と、変更履歴の記録による抑止力を活用。
2.  **Discord連携の深化**
    - Botによるメンバーリストの自動取得、リマインド、予定の自動投稿。
    - 認証が必要な管理者(Owner)のみDiscord OAuthを使用。
3.  **Googleカレンダーとの高度な同期**
    - メンバーの空き時間をFreeBusy APIで一括取得（自動で「×」を設定）。
    - 活動確定時にカレンダーへ予定を自動書き込み。
4.  **自動活動日決定 (Auto Scheduling)**
    - 「週N回」「特定の曜日を優先」などのルールに基づき、最適な活動日をシステムが提案・確定。

---

## 3. 主要機能
- **Availability Board (MUI採用)**: 月単位/週単位でのメンバー参加可否の一覧表示。
- **2層構造の認証**: 一般ユーザーは「ゆるい識別」、カレンダー連携ユーザーは「OAuth識別」。
- **自動確定 & 通知**: 確定タイミングでのDiscord投稿、前日のリマインド（チャンネル & DM）。
- **管理者機能**: 過去日付の編集ロック解除、グループ設定、メンバー管理（Discord同期）。
- **広告運用**: Google AdSenseによる収益化、将来的な有料プラン（広告非表示）の布石。

---

## 4. 技術スタック・アーキテクチャ
### モノレポ構成
- **Backend**: Ruby on Rails 7.1 (API Mode) + PostgreSQL + Sidekiq/Redis.
- **Frontend**: React 18 (Vite) + MUI + Tailwind CSS + TanStack Query.
- **Bot**: Node.js (discord.js).
- **Infrastructure**: Docker Compose (開発/本番) + ConoHa VPS (初期2GBプラン).

### 認証・識別システム
- **Session管理**: Google/Discord OAuth連携済みユーザーのみHttpOnly Secure Cookieによるセッション管理を行う。
- **データモデル**: 
    - `users`: Discord ID、Google連携情報を保持。
    - `availabilities`: 日付ごとのステータス(1, 0, -1, null)を管理。
    - `availability_logs`: 誰がいつどこから変更したかの履歴（抑止力）。

## 6. 特筆すべき設計判断
- **プライバシー配慮**: Googleカレンダーの予定タイトルや詳細は取得せず、空き状況（boolean）のみをキャッシュする。
- **国際化**: 内部データを数値で持ち、ロケールによって「○△×」と「✓?✗」を切り替える設計。
- **運用コストの最小化**: Sidekiq-cronによるバッチ処理と、ConoHa VPS 1台に全コンテナを集約する低コスト構成。
- **退会対応**: データの整合性を守るため、退会ユーザーは匿名化して履歴を保持（Ownerのみ閲覧可）。

---

## 7. リスクと対策
- **なりすましリスク**: 変更履歴（UA, IP地域情報）の記録と、Google連携済みユーザーへのロック（OAuth必須化）で対応。
- **API制限**: FreeBusy APIの一括取得と15分キャッシュによりGoogle APIのレート制限を回避。
- **広告収益**: 初期はVPSコストを広告で賄い、将来的にグループ課金モデル（1人が払えば全員広告消える）への移行を検討。
