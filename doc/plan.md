# プロジェクト計画書

## 概要

小規模グループ（最大約20名）向けのスケジュール管理ツールを開発する。メンバーの参加可否を可視化し、活動日の自動・手動設定、Discord および Google カレンダーとの連携機能を提供する。広告収入による運用を前提とし、低コスト構成で実現する。

## プロジェクトフェーズ

### フェーズ 1: 基盤構築（2〜3週間）

- Ruby on Rails プロジェクトのセットアップ（API モード）
- React フロントエンドプロジェクトのセットアップ
- データベース設計とマイグレーション作成
- ユーザー認証基盤の実装（Devise + JWT）
- CI/CD パイプラインの構築

### フェーズ 2: コア機能開発（3〜4週間）

- グループ管理機能（作成、招待、メンバー役割管理）
- Availability_Board の基本 UI 実装
- 参加可否（○/△/×）の入力・保存機能
- コメント入力機能
- 参加可否の可視化（色分け、強調表示）

### フェーズ 3: 活動日設定機能（2週間）

- 手動での活動日設定
- Auto_Schedule_Rule の設定 UI と自動生成ロジック
- 活動日の表示と管理

### フェーズ 4: 外部連携（3〜4週間）

- Google カレンダー OAuth 2.0 連携
- 予定枠のみの取得とプライバシー保護ロジック
- Calendar_Sync による自動×設定
- Discord Bot の開発と通知機能
- リマインダー機能

### フェーズ 5: 広告・運用準備（1〜2週間）

- Google AdSense の導入
- 広告配置の最適化（UX を妨げない配置）
- パフォーマンス最適化
- セキュリティレビュー

### フェーズ 6: テスト・リリース（2週間）

- 統合テスト・E2E テスト
- ユーザー受け入れテスト
- 本番環境デプロイ
- 運用監視の設定

## マイルストーン

| マイルストーン | 目標時期 | 成果物 |
|---|---|---|
| M1: 基盤完成 | フェーズ1完了時 | 認証付き API + React アプリの骨格 |
| M2: コア機能完成 | フェーズ2完了時 | 参加可否の入力・可視化が動作 |
| M3: 活動日管理完成 | フェーズ3完了時 | 自動・手動の活動日設定が動作 |
| M4: 外部連携完成 | フェーズ4完了時 | Google カレンダー・Discord 連携が動作 |
| M5: リリース準備完了 | フェーズ5完了時 | 広告表示・パフォーマンス最適化完了 |
| M6: 正式リリース | フェーズ6完了時 | 本番環境で稼働開始 |

## リスクと対策

| リスク | 影響度 | 対策 |
|---|---|---|
| Google API の利用制限 | 中 | 予定の有無のみ取得しリクエスト数を最小化。キャッシュ活用 |
| Discord API の変更 | 低 | discord.js の安定版を使用。Bot の権限を最小限に設定 |
| 広告収入の不足 | 中 | 初期は無料枠のインフラを活用。ユーザー数に応じてスケール |
| プライバシー問題 | 高 | 予定の詳細を一切保存しない設計。定期的なセキュリティ監査 |
| 少人数チームでの開発遅延 | 中 | フェーズごとに MVP を定義し段階的にリリース |

## 成功指標

- グループ作成から参加可否入力まで3分以内で完了できること
- Availability_Board の表示が1秒以内であること
- Google カレンダー連携の設定が5ステップ以内で完了できること
- 月間サーバーコストが広告収入で賄える水準であること
